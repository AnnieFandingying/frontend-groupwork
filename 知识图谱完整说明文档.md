## 🎯 系统概述

### 什么是知识图谱？

前端技术知识图谱是一个**可视化的知识网络系统**，通过节点和连线展示前端技术之间的关系和层级结构。帮助学习者：
- 📊 **系统化学习** - 了解技术的完整体系
- 🔍 **快速定位** - 通过搜索找到特定知识点
- 🎯 **规划路径** - 根据层级规划学习顺序
- 🔗 **理解关联** - 看清技术之间的依赖关系

### 核心特性

- ✅ **三级难度** - 初阶、中阶、高阶逐步进阶
- ✅ **七层结构** - 从中心节点到六级节点的清晰层级
- ✅ **五种关系** - 基础依赖、框架生态、工具依赖、分类包含、跨域应用
- ✅ **颜色编码** - 不同层级和关系类型用颜色区分
- ✅ **实时搜索** - 快速定位任意节点
- ✅ **独立数据** - 三个级别的数据完全分离，互不影响

## 🏗️ 节点层级结构

### 统一层级系统

知识图谱使用统一的7层结构，不同层级的节点有不同的大小和颜色：

| 层级 | 名称 | 大小(val) | 颜色 | 分组名称 | 说明 |
|------|------|-----------|------|----------|------|
| **0** | 中心节点 | 40 | 🔴 红色 `#FF6B6B` | `center` | 知识图谱的核心 |
| **1** | 一级节点 | 20 | 🔵 青色 `#4ECDC4` | `level1` | 主要知识分类 |
| **2** | 二级节点 | 12 | 🟦 蓝色 `#45B7D1` | `level2` | 次级分类 |
| **3** | 三级节点 | 8 | 🟢 绿色 `#96CEB4` | `level3` | 具体知识点 |
| **4** | 四级节点 | 6 | 🟡 黄色 `#FFEAA7` | `level4` | 详细知识点 |
| **5** | 五级节点 | 5 | 🟠 橙色 `#DDA15E` | `level5` | 细节知识点 |
| **6** | 六级节点 | 4 | 🟤 棕色 `#BC6C25` | `level6` | 最细粒度 |

### 颜色系统

#### 初阶和中阶图谱（使用层级颜色）
节点颜色根据层级自动分配，便于识别节点在知识体系中的位置。

#### 高阶图谱（使用分组颜色+掌握度）
- **分组颜色**: 根据技术分类（foundation、framework、tool等）
- **掌握度渲染**: 
  - mastery > 80: 蓝色（已掌握）
  - mastery > 50: 橙红色（学习中）
  - mastery ≤ 50: 灰色（未学习）

### 节点数据结构

```typescript
interface Node {
  id: string;                    // 节点唯一标识
  label: string;                 // 显示名称
  group: string;                 // 分组（center/level1-6/core等）
  val: number;                   // 节点大小
  mastery?: number;              // 掌握度 (0-100)
  level?: 'beginner' | 'intermediate' | 'advanced'; // 难度级别
}
```

## 🔗 关系类型说明

### 五种关系类型总览

高阶知识图谱支持5种关系类型，每种用不同颜色表示：

| 关系类型 | 颜色 | 数量 | 占比 | 说明 |
|---------|------|------|------|------|
| 🟣 **分类包含** (category) | `#9C27B0` 紫色 | 84 | 65.6% | 大类到子项的分类关系 |
| 🟢 **框架生态** (ecosystem) | `#4CAF50` 绿色 | 18 | 14.1% | 框架及其配套工具/库 |
| 🟠 **跨域应用** (application) | `#FF5722` 橙色 | 10 | 7.8% | 技术在不同场景的应用 |
| 🟡 **工具依赖** (dependency) | `#FFC107` 黄色 | 9 | 7.0% | 工具之间的技术依赖 |
| 🔵 **基础依赖** (foundation) | `#2196F3` 蓝色 | 7 | 5.5% | 核心技术的基础组成 |

### 详细说明

#### 🔵 基础依赖 (foundation)
表示核心技术的基础组成部分，是技术体系的根基。

**示例**:
- `Frontend → HTML` - 前端三剑客之一
- `Frontend → CSS` - 前端样式基础
- `Frontend → JS` - 前端逻辑基础
- `JS → TS` - TypeScript是JavaScript的超集
- `Intermediate_Center → Vue_js` - 中阶中心连接Vue

#### 🟢 框架生态 (ecosystem)
框架及其配套的工具、库和解决方案。

**示例**:
- React生态: `React → Redux`, `React → RN`, `React → Taro`
- Vue生态: `Vue → Pinia`, `Vue → UniApp`
- Node生态: `Node → Express`, `Node → NestJS`
- 构建生态: `Vite → Vitest`

#### 🟡 工具依赖 (dependency)
工具之间的底层技术依赖，表示技术栈的分层结构。

**示例**:
- `Webpack → Babel` - Webpack使用Babel转译
- `Vite → Rollup` - Vite生产环境使用Rollup
- `Vite → Esbuild` - Vite开发环境使用Esbuild
- `WebGL → ThreeJS` - Three.js封装WebGL

#### 🟣 分类包含 (category)
大类到子项的分类关系，构建清晰的知识层级体系。**数量最多的关系类型**。

**示例**:
- `Frontend → BuildTool` (构建工具)
- `BuildTool → Webpack`, `BuildTool → Vite`
- `Vue_js → 核心概念`, `核心概念 → MVVM_模式`
- `TypeScript → 泛型`, `泛型 → 泛型函数`

#### 🟠 跨域应用 (application)
技术在不同场景、领域的具体应用。

**示例**:
- `SSR → Node` - 服务端渲染需要Node.js
- `PWA → ServiceWorker` - PWA使用Service Worker
- `Auth → JWT` - 认证使用JWT
- `Node → MongoDB` - Node.js连接MongoDB

### 连接数据结构

```typescript
interface Link {
  source: string;                // 源节点ID
  target: string;                // 目标节点ID
  type?: 'foundation' | 'ecosystem' | 'dependency' | 'category' | 'application';
}
```

## 💻 使用方法

### 1. 在组件中导入

```typescript
// 导入高阶图谱
import { ADVANCED_GRAPH_DATA, RELATION_TYPES } from './constants';

// 导入初阶和中阶图谱
import { BEGINNER_GRAPH_DATA, INTERMEDIATE_GRAPH_DATA } from './graphLevels';
```

### 2. 单一级别使用

```vue
<template>
  <!-- 初阶图谱 -->
  <KnowledgeGraph :data="BEGINNER_GRAPH_DATA" />
  
  <!-- 中阶图谱 -->
  <KnowledgeGraph :data="INTERMEDIATE_GRAPH_DATA" />
  
  <!-- 高阶图谱 -->
  <KnowledgeGraph :data="ADVANCED_GRAPH_DATA" />
</template>
```

### 3. 动态切换级别

```vue
<script setup lang="ts">
import { ref, computed } from 'vue';
import { ADVANCED_GRAPH_DATA } from './constants';
import { BEGINNER_GRAPH_DATA, INTERMEDIATE_GRAPH_DATA } from './graphLevels';

const currentLevel = ref<'beginner' | 'intermediate' | 'advanced'>('beginner');

const currentGraphData = computed(() => {
  switch (currentLevel.value) {
    case 'beginner': return BEGINNER_GRAPH_DATA;
    case 'intermediate': return INTERMEDIATE_GRAPH_DATA;
    case 'advanced': return ADVANCED_GRAPH_DATA;
  }
});

function switchLevel(level: 'beginner' | 'intermediate' | 'advanced') {
  currentLevel.value = level;
}
</script>

<template>
  <div class="level-switcher">
    <button @click="switchLevel('beginner')" :class="{ active: currentLevel === 'beginner' }">
      🎯 初阶
    </button>
    <button @click="switchLevel('intermediate')" :class="{ active: currentLevel === 'intermediate' }">
      🚀 中阶
    </button>
    <button @click="switchLevel('advanced')" :class="{ active: currentLevel === 'advanced' }">
      ⭐ 高阶
    </button>
  </div>
  
  <KnowledgeGraph :data="currentGraphData" />
</template>
```

### 4. 节点颜色渲染

```vue
<script setup lang="ts">
// 初阶/中阶：使用层级颜色
const getNodeColor = (node: Node) => {
  if (currentLevel.value !== 'advanced') {
    // 层级颜色系统
    const colors = {
      center: '#FF6B6B',
      level1: '#4ECDC4',
      level2: '#45B7D1',
      level3: '#96CEB4',
      level4: '#FFEAA7',
      level5: '#DDA15E',
      level6: '#BC6C25'
    };
    return colors[node.group] || '#999';
  } else {
    // 高阶：使用掌握度
    if (node.mastery > 80) return '#2196F3'; // 蓝色
    if (node.mastery > 50) return '#FF5722'; // 橙红色
    return '#999'; // 灰色
  }
};
</script>
```

## 🛠️ 维护与更新

### 更新初阶图谱

初阶图谱目前保持稳定，如需更新：

1. 编辑Excel文件 `前端技术知识图谱(1).xlsx`（前434行）
2. 重新运行生成脚本
3. 检查 `graphLevels.ts` 中的 `BEGINNER_GRAPH_DATA`

### 更新中阶图谱

中阶图谱可以随时从Excel更新：

```bash
cd scripts

# 1. 解析Excel生成JSON
python convert_intermediate_excel.py

# 2. 将JSON转换为TypeScript代码
python generate_ts_code.py

# 3. 替换graphLevels.ts中的中阶数据
python replace_intermediate.py
```

**注意**: 
- ✅ 确保Excel文件路径正确
- ✅ 初阶数据完全不受影响
- ✅ 自动验证TypeScript语法

### 更新高阶图谱

直接编辑 `constants.ts` 中的 `ADVANCED_GRAPH_DATA`:

```typescript
export const ADVANCED_GRAPH_DATA: GraphData = {
  nodes: [
    { id: 'Frontend', label: '前端', group: 'core', val: 40, mastery: 90, level: 'advanced' },
    // ... 添加或修改节点
  ],
  links: [
    { source: 'Frontend', target: 'HTML', type: 'foundation' },
    // ... 添加或修改连接
  ]
};
```

**要求**:
- 所有节点必须有 `level: 'advanced'`
- 所有连接建议指定 `type` (5种之一)
- 保持数据结构的一致性

## ❓ 常见问题

### Q1: 三个级别的数据会互相影响吗？
**A**: 不会。初阶和中阶在 `graphLevels.ts`，高阶在 `constants.ts`，完全独立。

### Q2: 如何查看某个节点属于哪个层级？
**A**: 使用 `python scripts/view_node_levels.py intermediate` 查看所有层级。

### Q3: 更新中阶数据会影响初阶吗？
**A**: 不会。替换脚本只修改 `INTERMEDIATE_GRAPH_DATA` 部分，`BEGINNER_GRAPH_DATA` 完全不受影响。

### Q4: 高阶图谱的5种关系类型在初中阶有吗？
**A**: 初阶和中阶主要使用 `foundation` 和 `category` 两种关系，高阶才有完整的5种。

### Q5: 节点颜色是怎么确定的？
**A**: 
- 初阶/中阶：根据层级（level1-6）自动分配7种颜色
- 高阶：根据掌握度（mastery）分配蓝/橙/灰色
